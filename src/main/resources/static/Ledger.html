고정지출 관리
개발 예정
월세, 통신비, 구독 서비스 등 매달 정해진 날짜에 자동 출금되는 고정 지출 항목을 등록하고 관리합니다.
등록된 규칙에 따라 스케줄러가 자동으로 가계부 영수증을 생성하고 연동 자산 잔액을 차감합니다.

FixedExpenseRuleDTO 필드 구조

rule_id: int 규칙 식별자 (PK, AUTO_INCREMENT)
amount: long 정기 출금액 (예: 360,000 → 월세)
base_date: String 최초 기준일 (예: 2026-05-10)
p_value: int 주기 수치 (예: 1)
p_unit: String 주기 단위 (DAY / WEEK / MONTH)
💰
정기수입 관리
개발 예정
월급, 용돈, 정기 이자 등 규칙적으로 입금되는 수입 항목을 등록합니다.
설정된 주기마다 스케줄러가 자동으로 수입 영수증을 생성하고 연동 자산 잔액을 증가시킵니다.

RegularIncomeRuleDTO 필드 구조

rule_id: int 규칙 식별자 (PK, AUTO_INCREMENT)
amount: long 정기 입금액 (예: 500,000 → 월급)
base_date: String 최초 입금 기준일
p_value: int 주기 수치
p_unit: String 주기 단위 (DAY / WEEK / MONTH)
📊
변동지출 관리
개발 예정
가스비, 전기세 등 발생 주기는 일정하지만 금액이 매달 달라지는 항목을 관리합니다.
규칙이 발동하면 PENDING(미확정) 상태의 영수증이 생성되고, 사용자가 실제 금액을 입력하면 CONFIRMED(확정)으로 전환됩니다.

VariableExpenseReceiptDTO 필드 구조

rule_id: int 부모 규칙 ID (FK → variable_expense_rule)
amount: long 사용자 확정 금액 (PENDING 시 0)
transaction_date: String 실제 결제일
status: String "PENDING" / "CONFIRMED" / "SKIPPED"
